# Bideo 프로젝트 AI 모델 적용 추천 5가지

## 1. 작품 자동 태깅·카테고리 분류 (Work 도메인)
- **위치**: `WorkService.registerWork()` / `updateWork()` 시점
- **모델**: Vision 모델 (CLIP, BLIP-2 또는 GPT-4V)
- **기능**: 업로드된 영상/이미지를 분석해 장르·스타일·색감·주제 태그를 자동 생성, 카테고리 자동 분류
- **효과**: 사용자 입력 부담 ↓, 검색 정확도 ↑, 추천 시스템 데이터 풍부화

## 2. 개인화 추천 시스템 (Main + Gallery)
- **위치**: 메인 페이지 피드, 작품 상세 하단 "관련 작품"
- **모델**: 임베딩 기반 추천 (Sentence-BERT + 협업 필터링) 또는 Two-Tower
- **기능**: 사용자의 좋아요·찜·시청 기록 + 작품 임베딩으로 유사 작품 추천
- **효과**: 체류 시간 ↑, 작품 노출 다양성 ↑ — Redis에 임베딩 캐싱하면 응답 속도 보장

## 3. 유해 콘텐츠·저작권 위반 자동 검수 (Admin + Work 등록)
- **위치**: 작품 업로드 직후 비동기 큐 + Admin 신고 처리
- **모델**: NSFW 분류기 + 워터마크/표절 탐지 + 텍스트 모더레이션 LLM
- **기능**: 부적절 이미지/영상 자동 차단, 댓글 욕설·스팸 필터링, 표절 의심 작품 플래그
- **효과**: Admin 수동 검수 부담 대폭 감소, 플랫폼 신뢰도 ↑

## 4. AI 채팅 어시스턴트 / 작품 설명 자동 생성 (Message + Work)
- **위치**: `work-register` 페이지 작품 설명 자동 작성 버튼 + 고객문의 챗봇
- **모델**: LLM (Claude Haiku / GPT-4o-mini) — 비용 효율 모델
- **기능**:
  - 이미지 → 작품 설명·제목 자동 초안 생성
  - 사용자 FAQ·이용 안내 챗봇 (경매 방법, 결제 등)
- **효과**: 창작자 등록 마찰 ↓, CS 비용 ↓

## 5. 경매 시세 예측 / 공모전 심사 보조 (Auction + Contest)
- **위치**: `auction.html` 시작가 추천 + Contest 1차 스크리닝
- **모델**:
  - 회귀 모델 (XGBoost): 작가 이력·작품 특성·과거 낙찰가로 적정가 예측
  - LLM 평가자: 공모 주제 적합성 점수화
- **기능**: 출품자에게 시작가 가이드 제공, 공모전 다수 출품작 1차 자동 평가
- **효과**: 경매 활성화, 공모전 운영 효율화

## 적용 우선순위 제안

| 순위 | 기능 | 난이도 | ROI |
|------|------|--------|-----|
| 1 | 자동 태깅 | 낮음 | 높음 (검색·추천 기반) |
| 2 | 콘텐츠 모더레이션 | 중간 | 높음 (리스크 회피) |
| 3 | 작품 설명 자동 생성 | 낮음 | 중간 |
| 4 | 개인화 추천 | 높음 | 매우 높음 |
| 5 | 시세 예측 | 높음 | 중간 |

## 아키텍처 제안
- 별도 `ai` 패키지 신설
- `AiService` 인터페이스로 추상화
- 외부 API 호출은 `@Async` + Redis 캐싱으로 응답 지연 방지
- MyBatis 트랜잭션과 분리해 비동기 처리 권장
